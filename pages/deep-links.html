Deep Links & UTM Contracts

Not separate pages but URL patterns that trigger behavior

URL Patterns

/?book=1
Auto-opens 4-step booking dialog on homepage
Source: General deep-link (shareable) PostHog: booking_started
/?book=1&utm_source=gmb
Opens booking dialog + acquisition source = gmb
Source: Google Maps / GMB action button PostHog: source tracked
/?book=1&utm_source=walkin
Opens booking dialog + acquisition source = walkin
Source: In-store QR code posters PostHog: source tracked Trigger: QR Scan
/?book=1&service=[slug]
Opens dialog with service pre-selected in Step 3
Source: Service page "Book This" buttons PostHog: service pre-selection
/?book=1&leadId={id}
Opens dialog with lead context linked (post-lead-capture redirect)
Source: After /book form submit Lead → booking conversion tracked
/book?utm_source=meta&utm_campaign=X&utm_content=Y
Meta ad landing page with full campaign tracking
Source: Meta/Instagram ads only Meta Pixel + CAPI + PostHog Paid Traffic Only

Pre-Auth Preservation

Before Google OAuth redirect, all query parameters are stored in sessionStorage. After the OAuth callback, they're read back to restore context — opening the booking dialog, assigning acquisition source, and linking the lead.

Stored parameters:

book utm_source utm_campaign utm_content leadId service

Deep Link Flow — User Journey

1
User clicks link (GMB, QR, ad, or shared URL with query params)
2
Homepage loads — detects ?book=1 and other params
3
Params saved to sessionStorage (survives OAuth redirect)
4
Booking dialog opens automatically with pre-filled context (service, lead, source)
5
If auth required → OAuth redirect → callback reads sessionStorage → dialog re-opens
6
Analytics fire — PostHog event with source attribution, conversion tracked
