4.1 Sign In

/sign-in
TitleSign In
PurposeSingle entry point for authentication via Google OAuth
RenderingSSR
SEOrobots: noindex, nofollow
LayoutCentered card on gradient/brand background, Royal Glow logo above card

UI Components

  • Royal Glow logo — centered above the card
  • Card heading: "Sign in to Royal Glow"
  • Subheading: "Use your Google account to continue"
  • "Sign in with Google" button — full-width, Google branded
  • Legal text: "By signing in, you agree to our Privacy Policy and Terms of Service"
  • No email/password fields (Google OAuth only)

States & Behavior

  • Idle: Button active, ready for click
  • Redirecting: Button disabled with spinner — "Redirecting to Google..."
  • Error: "Sign-in failed. Please try again." with retry button (e.g., popup blocked)

Pre-Redirect: sessionStorage Preservation

  • Stores book=1 flag (if user came from booking intent)
  • Preserves all UTM fields: utm_source, utm_medium, utm_campaign, utm_content, utm_term
  • Stores leadId — links anonymous lead to authenticated user
  • Stores service slug — pre-selects service after auth completes
  • Data survives OAuth redirect → read on /onboarding or homepage return

Responsive

  • Mobile: Card full-width with bottom padding
  • Desktop: Card centered, max-width 400px, decorative background

4.2 Welcome Setup

/onboarding
TitleWelcome to Royal Glow
PurposeFirst-time user data collection — name, phone, DOB, gender, consent
RenderingSSR (auth-gated, shows only for users without a completed profile)
SEOrobots: noindex, nofollow
RedirectIf profile already complete → redirect to / immediately

UI Components

  • Welcome header: "Welcome to Royal Glow, [First Name]! 👑"
  • Name — prefilled from Google, editable text input with label
  • Email — prefilled from Google, read-only (greyed out, disabled)
  • Phone — tel input, validated Indian mobile format (+91), pre-populated from Google People API if available
  • Date of Birth — date picker, DD/MM/YYYY format (Indian standard)
  • Gender — select dropdown: Male / Female / Other / Prefer not to say
  • Submit button: "Let's Go!" — primary, full-width

Consent Section — "Privacy & Preferences"

"I agree to the Privacy Policy..." (links to /privacy) REQUIRED
"Help us improve Royal Glow — allow anonymous usage analytics" (optional)
"Send me offers, updates & promotions via email and notifications" (optional)

On Submit

  • POST /api/onboarding/complete — saves profile data
  • Acquisition source assigned from sessionStorage first-touch context (organic / gmb / walkin / meta_ad lead)
  • Consent choices written to rgss_cookie_consent in localStorage
  • Cookie consent banner suppressed for categories already consented to
  • Meta CAPI: CompleteRegistration event fired (server-side)
  • Redirect to / — if book=1 in sessionStorage, homepage auto-opens booking dialog

States

  • Validation Errors: Inline per field — phone format, DOB required, privacy consent required
  • Submitting: Button spinner, all fields disabled
  • Success: Instant redirect to / (no success page — seamless transition)

Analytics Events

  • PostHog: onboarding_completed with properties: gender, has_phone, consent_analytics, consent_marketing
  • Meta Pixel: CompleteRegistration (browser-side)
  • Meta CAPI: CompleteRegistration (server-side, deduped with Pixel)
