PENDING → CONFIRMED → IN_PROGRESS → COMPLETED
PENDING → REJECTED (with reason)
PENDING / CONFIRMED → CANCELLED (free if >4h before)
CONFIRMED → NO_SHOW (noshow_count++)

📋 Booking Dialog Steps

Step Title Description Key Actions
1 Date & Time Calendar picker + available time slots grid Select date, pick slot, auto-branch selection
2 Categories Salon / SPA toggle with multi-select categories Toggle type, select one or more categories
3 Services Per-category service list with prices & durations Select services, SPA 60/90 min toggle
4 Confirmation Booking summary with ID, total, gems balance Review, submit booking, view receipt

⚠️ Cancellation Rules

  • ✅ Free cancellation if > 4 hours before appointment
  • ⚠️ Tagged in CRM as "late cancellation" if < 4 hours
  • ❌ No-show count incremented if customer doesn't arrive
  • 🚫 At 4 no-shows: future bookings require manager approval

🛠 Booking States & Error Handling

Expand each section to see state details, UI behavior, and error handling.

PENDING — Initial booking state

Description: Booking has been submitted by customer, awaiting admin confirmation.

PropertyValue
TriggerCustomer completes Step 4 and submits
UI StateAmber "Pending" badge on booking card
NotificationsPush to admin; email confirmation to customer
Customer actionsCancel (free), view details
Admin actionsApprove (CONFIRMED), Reject (REJECTED)
Auto-transitionNone (requires admin action)

Error scenarios:

  • Slot already taken → show "Slot unavailable, please choose another time"
  • Network failure → retry with exponential backoff, show toast
  • User at 4+ no-shows → booking flagged for manager approval
CONFIRMED — Admin approved

Description: Admin has approved the booking. Customer will receive a confirmation notification.

PropertyValue
TriggerAdmin clicks "Approve" on booking
UI StateGreen "Confirmed" badge, add to calendar option
NotificationsPush + email to customer with details
Customer actionsCancel (free if >4h), reschedule
Admin actionsCheck-in (IN_PROGRESS), Cancel, mark No-Show
Reminders24h before, 2h before appointment

Error scenarios:

  • Late cancellation (<4h) → tagged in CRM, no penalty first time
  • Double-booking conflict → admin alerted, must resolve manually
IN_PROGRESS — Customer checked in

Description: Customer has arrived and been checked in by front desk.

PropertyValue
TriggerAdmin clicks "Check-in" at front desk
UI StateBlue "In Progress" badge, service timer shown
NotificationsInternal notification to assigned staff
Customer actionsView progress (read-only)
Admin actionsComplete (with invoice), add extra services
Duration trackingTimer starts from check-in timestamp
COMPLETED — Service finished

Description: Service has been completed, invoice generated, gems awarded.

PropertyValue
TriggerAdmin marks "Complete" after service is done
UI StatePurple "Completed" badge, invoice available
NotificationsInvoice email + gems earned notification
Customer actionsView invoice, leave review, rebook
GemsAwarded based on total spend (1 gem per 10 AED)
Follow-upFeedback request after 24h
REJECTED — Admin declined

Description: Booking was rejected by admin with a reason provided.

PropertyValue
TriggerAdmin clicks "Reject" with reason
UI StateRed "Rejected" badge with reason displayed
NotificationsPush + email to customer with rejection reason
Customer actionsView reason, rebook with different options
Common reasonsSlot conflict, service unavailable, staff leave
CANCELLED — Booking cancelled

Description: Booking was cancelled by customer or admin.

PropertyValue
TriggerCustomer or admin initiates cancellation
UI StateGrey "Cancelled" badge with timestamp
Free cancellation> 4 hours before appointment time
Late cancellation< 4 hours — tagged in CRM
Customer actionsRebook
CRM impactlate_cancellation_count incremented if <4h
NO_SHOW — Customer didn't arrive

Description: Customer didn't show up for their confirmed appointment.

PropertyValue
TriggerAuto-marked after appointment end time passes (or admin marks)
UI StateAmber "No Show" badge
NotificationsInternal alert to admin; no customer notification
CRM impactnoshow_count++ on customer profile
ThresholdAt 4 no-shows: future bookings require manager approval
RecoveryCustomer can still rebook but may be flagged

Escalation:

  • 1-3 no-shows: Warning shown to customer on next booking
  • 4+ no-shows: Booking goes to manager approval queue
  • Manager can override or block customer

🚀 API Endpoints

Method Endpoint Description Auth
GET /api/bookings/slots Get available time slots for a date + branch Public
GET /api/bookings/categories List service categories (Salon/SPA) Public
GET /api/bookings/services List services by category with prices Public
POST /api/bookings Create new booking (submit wizard) Customer
GET /api/bookings/:id Get booking details Customer/Admin
PATCH /api/bookings/:id/status Update booking status (admin only) Admin
DELETE /api/bookings/:id Cancel booking Customer/Admin
GET /api/bookings/my List customer's bookings Customer

📊 Status Transition Rules

From To Triggered By Conditions
PENDING CONFIRMED Admin approve No conflicts, staff available
PENDING REJECTED Admin reject Reason required
CONFIRMED IN_PROGRESS Admin check-in Within 30min of appointment
IN_PROGRESS COMPLETED Admin complete Invoice generated
PENDING CANCELLED Customer/Admin Always free if >4h before
CONFIRMED CANCELLED Customer/Admin CRM flag if <4h before
CONFIRMED NO_SHOW System/Admin After appointment end time
