Automatic Alerts
  • Stale alert (48h): Push notification to receptionist if no status change for 48 hours
  • Auto-Won: When booking.status = 'completed', lead auto-transitions to Won
  • Source attribution: Revenue tracked back to original lead source (Meta ROAS)

⚠️ No-Show Tier Progression

No-Show Tier Details
TierTagConsequenceRecovery
1stno_show_1Warning SMS/push sentComplete next = reset to 0
2ndno_show_2Phone call + manager notified2 consecutive = tier 1
3rdno_show_3₹500 deposit required + phone confirm 24h before3 consecutive = remove deposit
4thno_show_4₹1,000 deposit + owner reviewOwner discretion
5thno_show_5Blocked from online booking — phone onlyOwner manual reset
Implementation Notes
  • noshow_count stored on customer row (integer)
  • Tag auto-assigned: no_show_{count} (replaces previous tier tag)
  • Recovery tracked via consecutive_completed_count (resets on new no-show)
  • Block enforcement: booking API returns 403 if noshow_count ≥ 5
  • Owner override: PATCH /api/admin/customers/[id] { noshow_count: 0 }
