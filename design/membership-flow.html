Membership Rules

  • No rollover — unused hours are forfeited at expiry
  • No pause — membership runs continuously once activated
  • No transfer — membership cannot be moved between branches
  • Branch-locked — valid only at the branch of purchase
  • No gems — loyalty gems are not earned on membership services
  • No refund on cancellation — cancelled memberships forfeit remaining hours

Membership States & Error Handling

Each collapsible section details a state or error scenario within the membership lifecycle.

Active Active State — Normal Operation

Condition: status = 'active' AND remaining_hours > 0 AND expires_at > now()

Behaviour:

  • Sessions can be recorded against this membership
  • Progress bar shows hours used vs total
  • Customer can view session history on /membership
  • Admin can record sessions and view full detail on /admin/memberships/[id]
Low Low Balance Alert — Less than 60 minutes

Condition: remaining_hours < 1 (i.e., fewer than 60 minutes left)

Trigger: After any session recording that brings balance below 60 min.

Actions:

  • Push notification to customer: "Your SPA membership has less than 1 hour remaining"
  • Email with remaining balance and booking CTA
  • UI badge changes to amber warning in customer view
  • Progress bar turns red (.progress-fill.low)
T-30 Expiry Warning — 30 days before expiry

Condition: expires_at - now() <= 30 days

Actions:

  • Email: "Your Royal Glow SPA membership expires in 30 days"
  • Push notification with remaining hours summary
  • Triggered by QStash scheduled job (daily check)
T-7 Urgent Warning — 7 days before expiry

Condition: expires_at - now() <= 7 days

Actions:

  • Urgent email: "Only 7 days left — book your remaining hours now!"
  • Push notification with urgency flag
  • Customer UI shows amber warning banner
  • Admin dashboard highlights expiring memberships
T-1 Last Chance — 1 day before expiry

Condition: expires_at - now() <= 1 day

Actions:

  • Final email: "Your membership expires TOMORROW — last chance to use your hours"
  • Push notification with highest priority
  • Customer UI shows red warning banner
  • SMS sent as backup (if enabled)
Expired Expired State — Membership ended

Condition: status = 'expired' (set by pg_cron when expires_at <= now())

Transition: Automated via pg_cron job running daily at midnight IST.

Actions:

  • Status updated to 'expired' in database
  • Unused hours are forfeited (no rollover)
  • Final email sent with summary and renewal CTA
  • Sessions can no longer be recorded
  • Customer sees "Membership Expired" with renewal button
  • Admin sees greyed-out card with expiry date
Cancelled Cancelled State — Manual cancellation

Condition: status = 'cancelled' (set by manager/owner action)

Who can cancel: Manager or Owner role only (not staff, not customer).

Required:

  • Cancel reason must be provided (free text, required field)
  • Confirmation dialog: "This action is irreversible. No refund will be issued."

Actions:

  • cancelled_at timestamp recorded
  • cancel_reason stored
  • No refund issued — remaining hours forfeited
  • Customer notified via email
  • Sessions can no longer be recorded
Validation Record Session — Insufficient Hours Error

Scenario: Admin tries to record a session longer than remaining hours.

Validation Rule: duration_min <= remaining_hours * 60

Error Response:

Error: Cannot record session. Requested duration (120 min) exceeds remaining balance (45 min). Please reduce duration or record partial session.

UX Handling:

  • Toast notification with error message
  • Duration input highlighted in red
  • Suggestion to record remaining available time
  • Membership status does not change on validation failure
Flow Session Recording — Step by step

Steps for admin recording a session:

  1. Select Service: Choose from available SPA services (massage, facial, body treatment, etc.)
  2. Enter Duration: Minutes spent (validated against remaining hours)
  3. Assign Staff: Select the therapist/staff who performed the service
  4. Confirm: Review summary → "Record Session" button
  5. Result: Hours deducted, session logged with timestamp, progress bar updated

Post-recording checks:

  • If remaining < 60 min → trigger low balance alert
  • If remaining = 0 → show "All hours used" state
  • Ably real-time update to customer app

API Endpoints

POST /admin/memberships
Create a new membership for a customer. Requires admin/manager role.
Body: { customer_id, tier, total_hours, price, start_date, payment_method }
Response: 201 { membership } | 400 { error } | 403 { unauthorized }
GET /admin/memberships/[id]
Get full membership detail with session history. Requires admin role.
Response: 200 { membership, sessions[] } | 404 { not_found }
POST /admin/memberships/[id]/record-session
Record a session against an active membership. Validates hours balance.
Body: { service, duration_min, staff_id, notes? }
Response: 200 { session, updated_membership } | 422 { insufficient_hours }
POST /admin/memberships/[id]/cancel
Cancel a membership. Manager/Owner only. Requires cancel_reason.
Body: { cancel_reason }
Response: 200 { membership } | 403 { only_manager_owner }
GET /membership
Customer views their own active membership. Auth required (customer token).
Response: 200 { membership, sessions[] } | 404 { no_active_membership }

Business Rules & Constraints

RuleEnforcementError if violated
No rollover pg_cron sets status='expired' at midnight N/A (automated)
No pause No pause endpoint exists; no pause status N/A (no UI affordance)
No transfer between branches branch_id FK on membership; session recording validates branch match 422: branch_mismatch
Branch-locked RLS policy filters by staff's current branch 403: wrong_branch
No gems on membership Gems service skips if payment_source = 'membership' N/A (logic gate)
Session duration ≤ remaining Server-side check before INSERT 422: insufficient_hours
Cancel requires reason cancel_reason required in request body 400: cancel_reason_required
Cancel = manager/owner only Role check middleware 403: only_manager_owner
