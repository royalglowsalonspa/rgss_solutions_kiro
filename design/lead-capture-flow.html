🚫 /book Page Design Rules

⚠️
Cold traffic optimization — every element is intentional:
  • NO header navigation — zero distractions
  • NO footer links — only address + phone number
  • NO "Sign in" prompt — cold traffic bounces on auth screens
  • NO link from main site navigation — ever
  • Trust signals ABOVE the form (stars, reviews, city)
  • CTA says "Continue to Booking" NOT "Submit"
  • Phone field: +91 prefix auto-shown, 10-digit validation
  • Form fits above fold on most phones
  • Min touch target: 44px
  • Meta: robots: noindex, nofollow

Rate limiting: 5 requests per phone number per hour. Returns 429 Too Many Requests if exceeded. Prevents spam/abuse from bots.

🔗 UTM Parameter Handling

// On /book page load, extract UTMs from URL const params = new URLSearchParams(window.location.search); const utmData = { utm_source: params.get('utm_source'), // facebook | instagram utm_campaign: params.get('utm_campaign'), // campaign name utm_content: params.get('utm_content'), // ad creative ID fbp: getCookie('_fbp'), // Facebook pixel browser ID fbc: getCookie('_fbc') || params.get('fbclid') ? `fb.1.${Date.now()}.${params.get('fbclid')}` : null, // Facebook click ID }; // UTMs stored in hidden fields, submitted with form
⚡
Critical: _fbp and _fbc cookies are set by the Meta Pixel SDK. The page MUST load the Pixel script before form submission to ensure these cookies exist. If _fbc cookie is missing but fbclid URL param exists, construct it manually.

🟣 Meta Pixel + CAPI Dual Tracking

// Browser-side: Meta Pixel fires on form submit const eventID = `lead_${crypto.randomUUID()}`; fbq('track', 'Lead', { content_name: serviceInterested, content_category: 'salon_service', value: 0, currency: 'INR', }, { eventID: eventID }); // Server-side: CAPI fires in POST /api/leads handler await metaConversionsAPI.sendEvent({ event_name: 'Lead', event_id: eventID, // SAME ID for deduplication event_time: Math.floor(Date.now() / 1000), user_data: { ph: hashSHA256(phone), // Hashed phone fn: hashSHA256(firstName), fbp: fbpCookie, fbc: fbcCookie, }, action_source: 'website', event_source_url: 'https://royalglow.in/book', });
🔁
Deduplication: Meta uses event_id to deduplicate browser + server events. Both MUST send the same eventID string. If either is missing, Meta counts it once anyway — dual tracking is for reliability, not double-counting.

📊 Conversion Attribution Chain

Stage Event Source Links To
1. Page loads PageView Browser (Pixel) fbp + fbc cookies
2. Form submitted Lead Browser + CAPI lead.id + event_id
3. Dialog opens InitiateCheckout Browser (Pixel) leadId URL param
4. First sign-up CompleteRegistration Browser (Pixel) user.id created
5. Invoice paid Purchase CAPI (server only) lead.converted_booking_id → invoice.amount
💡
Purchase fires server-side only because it happens when the receptionist marks an invoice as paid — there's no customer browser session at that moment. The CAPI call includes the original fbp/fbc from the lead row for attribution.

📞 Receptionist Follow-up Flow

Trigger Action Channel
Lead created (status: new) Appears in CRM pipeline "New Leads" column Admin dashboard + Ably notification
Lead not booked within 30 min Receptionist gets nudge notification Push notification
Lead not booked within 4 hours Auto-WhatsApp follow-up (template) WhatsApp Business API
Lead stale > 48 hours ⚠️ Alert: "Stale lead — call immediately" Admin banner + SMS to receptionist
Lead booked (status: booked) Move to "Booked" column, link to booking CRM update
Lead lost (no contact after 7 days) Status → lost, added to retarget audience Meta Custom Audience sync
⏰
48h Stale Lead Alert: Background job runs every hour. If a lead with status 'new' or 'contacted' is older than 48 hours, it triggers an urgent alert. These are paid leads — every hour of delay reduces conversion probability.
